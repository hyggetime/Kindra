# 킨드라 Gemini Few-shot — 원문 추출·초안 (첨삭용)

랜딩 **영희 예시 리포트**(`src/components/ReportExampleYeonghui.tsx`)와 **지오 리포트**(`src/data/reports/jio.json`)에서, 모델에게 주입할 **분석 프롬프트 요소**를 뽑았습니다. 아래 블록을 복사해 마크다운 명문화하시면 됩니다.

## 공통 철학·한계 (두 사례 공통)

- **비단정**: "정답을 맞히는 탐정이 아니다", 구체 서사는 부모만 안다, 우리는 에너지·감정 밀도·태도의 **이면의 결**을 조심스럽게 읽는다. (지오 `philosophyBody` / 랜딩 카피와 동일 계열)
- **진단 거부**: 의학·임상 진단이 아님; 경향·관찰 제시. (지오 `footer.disclaimer` 계열)

## 사례 1에서 추출한 '분석 엔진' (영희 · 5장 시리즈)

### 입력 세계 (그림 1~5의 역할 — UI 카피 기준)

| 순서 | 시각·재료 특징 (모델 입력 힌트로 쓸 요약) |
|------|---------------------------------------------|
| 그림 1 | KFD 가족화 — 4인 한 테두리, 아이 쪽 고디테일(머리핀·귀걸이·속눈썹·신발), 성인은 얼굴 중심 |
| 그림 2 | HTP 맥락 인물화 — 나무 사이 소녀, 새싹에 물, **기저선**, 눈 감은 미소 |
| 그림 3 | 해변 군상 — 화면 구역 설계, 역할·동작 대비, 하늘·땅 꼼꼼 채움 |
| 그림 4 | 감정 카드 **메타 표상** — 카드 안에 다른 표정, 하트·역삼각형 필압 대비 |
| 그림 5 | 종이 인형 — 가위 오림, 2D→3D, 눈 감은 미소 반복 |

### 분석 차원 (섹션 라벨 패턴 — 그대로 프롬프트 키워드로 사용 가능)

1. **Line · Pressure · 선과 필압** — 균일 필압, Hammer(1958) 인용, 도구=감정 언어.
2. **Space · Ground · 공간 구성** — 전면 활용 vs 기저선, Hammer HTP.
3. **DAP · Figure Drawing · 인물 표현** — Koppitz(1968) 부속요소·변별력.
4. **Recurring Motif · 반복 표정** — 눈 감은 미소 = 내적 평온 서명.
5. **Narrative Cognition · 서사적 사고** — 나열이 아닌 장면 연출, 2D→3D 전환.
6. **Nurturing Motif · 양육 모티프** — 돌봄 주제의 세 가지 표현(물주기·인형·보호자).
7. **Meta-Representation · 메타 표상** — 그림 속 그림, ToM 연결(단정적 표현 완화).

### 부모 대면 (Hygge) 패턴

- **공간 설계자** — 스토리 있는 공간 구성 놀이.
- **과정의 칭찬** — 결과 칭찬 대신 과정 묘사.
- **세밀한 관찰자** — 디테일 포착 격려 + 놀이 연결.

### 색 스펙트럼

- 팔레트 라벨(흙·하늘·주황 등) + HEX — 시각 무드를 **데이터 줄**로 넘길 수 있음.

## 사례 2에서 추출한 '분석 엔진' (지오 · 단일 복합 시트)

### 입력 세계 (한 장 안의 이중 채널)

- **좌·우 분할**: 텍스트(형광 밑줄) vs 가족 이미지; 접힌 자국 = 문서형 표현.
- **KFD**: 세 사람 줄, 아이 중앙, 아빠 밀착/손잡음, 공통 눈 감은 미소·홍조.
- **DAP·상징**: 역할별 의상, 하트 층위, 말풍선 문구(사랑 순환), "서비스" 선물 모티프.
- **색**: 따뜻한 색 vs 부드러운 색 — **색만으로 정서 단정하지 않기** 명시.
- **내러티브**: 편지의 약속·감사와 그림의 화해·온기 — **가정 문제 단정 불가** + 부모 해석 최우선.

### 출력 구조 (JSON 필드 → 프롬프트 섹션 매핑)

| JSON 필드 | 프롬프트 섹션 제안 |
|-----------|---------------------|
| `analysisSections[].label` | 소제목 영·한 병기 |
| `analysisSections[].title` | 한글 헤드라인 |
| `analysisSections[].body` | 근거→가능 읽기→부모 확인 문장 |
| `summary` | 통합 한 줄 |
| `tags` | `#태그` 줄 |
| `hyggeTips` | 부모님께 실행 제안 |

## 멀티모달(5장 동시)용 시스템에 넣을 한 문단 (초안)

> 동일 요청에 첨부된 5개의 이미지는 업로드 순서대로 그림 1~5이다. 각 그림에 대해 시각적 사실을 먼저 적은 뒤, 5장을 횡단으로 비교하여 반복·대비 패턴을 찾는다. 임상적 진단명은 사용하지 않으며, 모든 해석 문장에는 근거가 된 그림 번호를 붙인다. 마지막에 비진단 안내를 포함한다.
